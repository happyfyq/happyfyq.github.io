超详细搭建属于自己的博客-Git+Node.js+Hexo

发表于 2017-05-22 | 分类于 搭建博客 | | 阅读

前言

我一直有记笔记的习惯，方便自己需要的时候回头看看，主要自己现在还是个菜鸟，需要经常的回顾知识。我在CSDN也有账号，也写过博客，但是界面不友好，就没太大兴趣继续写了，一个偶然的机会，看见别人的博客主页面很好，很清晰。于是乎也想着自己搭建一个自己的博客，创造属于自己的分享天地。Ok，就这样开启了搭建博客之路，在这里我把经验分享给大家，希望大家也多多分享知识，IT行业需要多多交流，不可闭门造车。OK，废话不多说

需要的环境

下载并安装msysgit

下载地址：https://git-for-windows.github.io/
安装方法：http://jingyan.baidu.com/album/e52e36154233ef40c70c5153.html?picindex=10

OK，以上所有内容仅限于在我们本地的修改，和使用。但是我们写博客的目的不仅限于自己看，还要分享给大家看，那么问题就来了，我们怎么样才能将我们本地博客搞到一个大家都可以用到的地方呢，DaMiss选择的是gitHub这个免费资源。还不知道gitHub是啥东东的小伙伴，可自行在百度搜索，一搜一大堆呢！OK，闲话少说，我们接着往下看…

首先我们需要在gitHub官网上注册自己的账号和新建自己的仓库。（已经有账号的小伙伴可自行忽略申请账号）

申请gitHub账号

官网地址：https://github.com/ 进行注册账号。
新建自己的仓库：注册账号之后，在页面的右上角有个+号，我们点开 new repository ，在这里着重强调一下仓库的名字的规则： yourgitHubname.gitHub.io这种格式的，否则会出现错误，比如DaMiss的gitHub账号是happyfyq ，那我的仓库名字就是happyfyq.github.io.其他默认选择，然后点击 create repository，ok完成!

创建密钥

创建密钥的目的是：SSH Keys，看你自己了，可以不配制，不配置的话以后每次对自己的博客有改动提交的时候就要手动输入账号密码，配置了就不需要了。

我们在设置密钥的时候，首先要创建git的用户名和密码,进去git bash(用户名和github一样，邮箱也是如此)

出现的问题以及解决办法

1.ERROR Deployer not found: git 或者 ERROR Deployer not found: github

解决：在cmd切换到博客主目录执行：npm install hexo-deployer-git - -save

记住这个save前面是两个 -

2.执行hexo d 的时候，一直出现提示you have to configure the depoment settings in _config.yml firtst!

解决：在站点配置文件里面配置deploy的时候：repo: git@github.com:happyfyq/happyfyq.github.io.git 这里的repo我一开始写成ssh了，改成repo就好了！

3.在修改配置文件_config.yml的时候，冒号后面一定要有空格，否则会出错，很多人都犯了这个错误。
